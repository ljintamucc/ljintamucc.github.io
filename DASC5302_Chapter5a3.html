Chapter 5.III K-Nearest Neighbors (KNN)¶

K-Nearest Neighbors (KNN) is a simple yet powerful and intuitive machine learning algorithm used for both classification and regression tasks.
It is a non-parametric, nonlinear, and instance-based learning method, meaning that it makes no assumptions about the underlying data distribution.
Instead, it relies directly on the training data to make predictions by measuring similarity between data points.

In KNN regression, the predicted value for a new input $x$ is obtained by averaging the responses of its $k$ nearest neighbors:

$$ \hat{f}_k(x) = \frac{1}{k} \sum_{i \in \mathcal{N}_k(x)} y_i $$

where $\mathcal{N}_k(x)$ denotes the set of indices corresponding to the $k$ nearest neighbors of $x$ in the training data.

The distance metric most commonly the Euclidean distance, but also alternatives such as Manhattan, Minkowski, or Mahalanobis distance is used to quantify the similarity between points.
For example, when $k = 3$, the algorithm identifies the three nearest points to $x$ and predicts the target value by averaging their corresponding responses $y_i$.

KNN operates under the principle of feature similarity: if two data points are close in the feature space, their outputs are likely to be similar. Hence, KNN assigns a value (for regression) or a label (for classification) to a new observation by examining its local neighborhood.

When to Use KNN¶

KNN performs well when:

  • The dataset is small to medium-sized (since the algorithm scales poorly with very large datasets).
  • There is limited prior knowledge about the data-generating process.
  • The relationship between predictors and response is nonlinear or complex, making parametric models unsuitable.
  • The data are well-scaled and normalized, ensuring that no feature dominates the distance computation.

It is often applied in:

  • Image recognition and computer vision tasks.
  • Recommendation systems (e.g., similarity-based retrieval).
  • Medical diagnostics and pattern recognition.

Key Considerations in KNN¶

  1. Choice of $K$ (Number of Neighbors)
    The parameter $k$ controls the trade-off between bias and variance:

    • A small $k$ (e.g., $k=1$) can lead to high variance and overfitting, since predictions rely heavily on individual noisy data points.
    • A large $k$ smooths the decision boundary, potentially causing underfitting.
    • A good rule of thumb is to use cross-validation to select $k$ that minimizes the prediction error.

  2. Distance Metric
    The choice of metric affects which points are considered neighbors:

    • Euclidean distance for continuous, well-scaled variables.
    • Manhattan distance for high-dimensional or sparse data.
    • Mahalanobis distance when features are correlated.
    • For categorical features, similarity measures like Hamming distance may be used.

  3. Feature Scaling
    Because KNN depends on distance calculations, feature normalization (e.g., minmax or z-score scaling) is essential.
    Without scaling, features with large numeric ranges can dominate the distance metric.

  4. Computational Efficiency
    KNN is a lazy learner, meaning that no explicit model is trained; predictions are made at query time.
    This leads to high computational cost for large datasets.
    Efficient data structures such as KD-trees, Ball trees, or approximate nearest neighbor (ANN) algorithms can help accelerate neighbor searches.

  5. Imbalanced Data
    In classification tasks, if one class dominates the dataset, KNN may be biased toward that class.
    Solutions include:

    • Weighted KNN, where neighbors closer to $x$ have higher influence.

    • Resampling or SMOTE to balance class distributions.

Advantages of KNN

  • Simple to understand and implement.
  • Naturally captures nonlinear relationships.
  • No explicit training phase the model is ready after data collection.
  • Works well with multi-modal data distributions.

Limitations

  • Computationally expensive for large datasets ($O(n)$ per query).
  • Sensitive to irrelevant or highly correlated features.
  • Performance deteriorates in high-dimensional spaces due to the curse of dimensionality.
  • Requires careful scaling and feature selection for optimal results.

Data Scaling¶

Before applying K-Nearest Neighbors (KNN), it is essential to preprocess the data to ensure the algorithm performs effectively. This preprocessing typically includes handling outliers, missing values, and feature scaling or normalization. Because the KNN algorithm is distance-based, unscaled features can distort the notion of "closeness" between data points. For example, if one feature has a range from 010,000 (e.g., income) and another from 010 (e.g., education level), the distance measure will be dominated by the larger-scale feature unless proper scaling is applied.

Key Preprocessing Steps¶

  1. Handling Missing Values
    Missing values can bias distance computations or cause errors in training.
    Common solutions include:

    • Mean or median imputation for continuous variables.
    • Mode imputation for categorical variables.
    • KNN-based imputation, where missing values are estimated using nearby points.

  2. Outlier Detection and Treatment
    Outliers can disproportionately influence distance-based methods.
    Techniques such as z-score filtering, IQR range filtering, or robust scaling can be used to reduce their impact.

  3. Feature Scaling / Normalization
    Scaling ensures that all features contribute equally to distance computations.
    Common approaches include:

    • MinMax Scaling: rescales features to the range $[0, 1]$ $$ x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}} $$

    • Z-score Standardization: centers data to mean $0$ and standard deviation $1$ $$ x' = \frac{x - \mu}{\sigma} $$

    • Robust Scaling: based on the median and interquartile range (IQR), effective when data contain outliers $$ x' = \frac{x - \text{median}(x)}{\text{IQR}(x)} $$

  4. Dimensionality Reduction (Optional)
    In high-dimensional spaces, distances between points become less meaningful this is known as the curse of dimensionality.
    Applying Principal Component Analysis (PCA) or feature selection can help reduce noise and improve KNN performance.
