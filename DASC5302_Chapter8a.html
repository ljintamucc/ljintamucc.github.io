Chapter 8 Classification¶

Introduction¶

Classification is one of the most fundamental tasks in machine learning and predictive modeling.
It is a supervised learning approach, meaning that the model is trained on labeled data—datasets in which the target outcome (or “class”) is already known.
The goal of classification is to assign a predefined class label to new, unseen data points based on their features or characteristics.

In simpler terms, classification helps a computer learn from examples so that it can correctly categorize new observations into one of several possible groups.

There are two main paradigms for classification:

  1. Model-Based Approach:
    Assumes that the data come from an underlying probabilistic model—for example, a joint distribution of features and labels.
    Once this distribution is estimated, we can make predictions by computing the probability that an observation belongs to each class.
    Examples include:

    • Logistic Regression
    • Linear/Quadratic Discriminant Analysis (LDA/QDA)
    • Naïve Bayes Classifier

  2. Model-Free Approach:
    Makes no assumptions about the underlying data distribution.
    Instead, classification rules are determined directly from the training data, often using distance, similarity, or decision boundaries.
    Examples include:

    • k-Nearest Neighbors (KNN)
    • Decision Trees and Random Forests
    • Support Vector Machines (SVM)

Based on the number of classes, classification tasks are categorized as:

  • Binary Classification:
    Two possible outcomes, such as
    “spam” vs. “not spam,”
    or “disease present” vs. “disease absent.”

  • Multiclass Classification:
    More than two possible categories, such as
    classifying flowers into setosa, versicolor, and virginica in the Iris dataset.

  • (Optional extension)
    Multilabel Classification:
    Each observation can belong to multiple classes simultaneously, such as tagging an image as both “indoor” and “people.”

Example Applications

Domain Description
Email Spam Detection Email providers such as Gmail classify incoming messages as spam, regular, or important based on content and sender patterns.
Medical Diagnosis Identifying whether medical images (e.g., mammograms or retinal scans) indicate the presence of disease.
Sentiment Analysis Classifying text (from reviews or social media) into positive, neutral, or negative sentiment.
Credit Scoring Assigning applicants to risk categories to evaluate their likelihood of repaying loans.
Image Recognition Recognizing handwritten digits (MNIST) or object categories (CIFAR-10).

Metrics to Measure Performance¶

Evaluating classification models requires multiple metrics since no single number can describe performance in all scenarios.
The most common metrics are accuracy, precision, recall, and the Kappa statistic.

1. Accuracy

$$ \text{Accuracy} = \frac{\text{Number of correct predictions}}{\text{Total number of predictions}} $$

Accuracy indicates how often the classifier’s predictions are correct overall.
However, it can be misleading for imbalanced datasets, where one class is much more frequent than others.

Example:
If 95% of patients are healthy and 5% have a disease, a model that always predicts “healthy” will have 95% accuracy but 0% recall for the disease class.

2. Kappa Statistic (Cohen’s Kappa)

The Kappa coefficient ($\kappa$) measures how much better a classifier performs compared to random guessing, taking into account the possibility of agreement by chance.

$$ \kappa = \frac{p_o - p_e}{1 - p_e} $$

where

  • $p_o$: observed agreement (accuracy),
  • $p_e$: expected agreement by chance.

Interpretation:

  • $\kappa = 0$: performance equals random guessing
  • $\kappa < 0$: worse than random
  • $\kappa = 1$: perfect agreement
  • $0.61 \le \kappa \le 0.80$: substantial agreement
  • $0.81 \le \kappa \le 1.00$: almost perfect agreement

Kappa is especially useful for imbalanced datasets and multi-class problems, providing a more robust measure than raw accuracy.

3. Precision (Positive Predictive Value)

$$ \text{Precision} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Positives}} $$

Precision measures how often the positive predictions are correct.
It is high when false positives are rare.

Example: In spam detection, high precision means that most of the emails flagged as spam are truly spam.

4. Recall (Sensitivity or True Positive Rate)

$$ \text{Recall} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Negatives}} $$

Recall measures the model’s ability to find all actual positives—how many true positive cases were successfully identified.

Example: In medical diagnosis, high recall means few diseased patients are missed.

5. F1 Score (Harmonic Mean of Precision and Recall)

$$ \text{F1} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} $$

The F1 score provides a balance between precision and recall and is particularly informative when dealing with uneven class distributions.

Choosing the Right Metric

  • Use accuracy when all classes are balanced and equally important.
  • Use precision when false positives are costly (e.g., flagging legitimate emails as spam).
  • Use recall when false negatives are costly (e.g., missing a disease case).
  • Use F1 or Kappa when you need a balanced assessment across multiple dimensions.

The Confusion Matrix

A confusion matrix is a table that summarizes the performance of a classification model by showing how well predicted labels match the true labels.

It compares actual (true) class values with predicted class values and helps identify which types of errors the model makes.

Confusion Matrix Structure

For a binary classification problem, the confusion matrix is typically represented as:

Predicted: Positive Predicted: Negative
Actual: Positive True Positive (TP) False Negative (FN)
Actual: Negative False Positive (FP) True Negative (TN)
  • True Positive (TP): Model correctly predicts the positive class
  • True Negative (TN): Model correctly predicts the negative class
  • False Positive (FP): Model incorrectly predicts positive (Type I error)
  • False Negative (FN): Model incorrectly predicts negative (Type II error)

If we have $K$ classes (e.g., Class 1, Class 2, Class 3), the confusion matrix looks like this:

Actual \ Predicted Class 1 Class 2 Class 3
Class 1 $TP_1$ $FP_{1,2}$ $FP_{1,3}$
Class 2 $FP_{2,1}$ $TP_2$ $FP_{2,3}$
Class 3 $FP_{3,1}$ $FP_{3,2}$ $TP_3$
  • Each row represents the actual class,
  • Each column represents the predicted class.
  • The diagonal entries ($TP_1, TP_2, TP_3, \ldots$) are correct predictions for each class.
  • Off-diagonal entries show misclassifications — how often one class is confused with another.
