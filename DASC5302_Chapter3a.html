Chapter 3 Linear regression and performance measure¶

3.1 Simple linear regression & correlation¶

Linear regression is to find a straight line to fit the data, to see how a response depends on some predictors (independent variables). The goal of simple linear regression is to fit a straight line to the data that best represents the relationship between the response and the predictor ( a straight line to minimize the prediction error). In Simple linear regression, there is only one predictor.

In a simple linear regression, there is only one predictor. The mathematical model:

$$y_i = \beta_0 + \beta_1 x_i + e_i$$

where $\beta_0$ is the intercept, $\beta_1$ is the slope and $e_i$ is the error term.

In machine learning workflows, simple linear regression is usually as the first supervised model for regression tasks, valued more for its interpretability and clarity than for raw predictive power.

Residulas¶

Residuals refer to the differences between the observed values of the dependent variable and the values predicted by the regression model. For the i-th observation : the vertical distance from the i-th observation to the regression line

$$ \hat {e}_i = y_i - \hat {y} $$

where $\hat {y}$ is the predicted value when $x$ is $x_i$.

Least squares method ( The straight line minimizes the total of all squared residuals. )
$$\min \sum_{i=1}^{n} \hat {e}^2_i = \min \sum_{i=1}^{n}(y_i-\hat{y}_i)^2$$

The solutions for the regression are:

$\hat {\beta}_1=\frac{\sum_{i=1}^{n}(x_i-\bar{x})(y_i-\bar{y})}{\sum_{i=1}^{n}(x_i-\bar{x})^2}$

$\hat {\beta}_0=\bar{y}-\beta_1\bar{x}$

Correlation¶

The Pearson correlation coefficient measures the strength and direction of the relationship between two quantitative variables. The sample correlation is

$$r=\dfrac{\sum_{i=1}^{n}(x_i-\bar{x})(y_i-\bar{y})}{\sqrt{\sum_{i=1}^{n}(x_i-\bar{x})^2\sum_{i=1}^{n}(y_i-\bar{y})^2}}$$

Comments:

$r$ is always between –1 and 1. The extreme value r = -1 or r = 1 occurs when the points in the scatterplot lie exactly along a straight line.

Correlation r measures only the linear relationship between two variables. Correlation does not describe curved relationships between variables, no matter how strong they are.

If there is a strong positive linear relationship between the variables, the correlation r is close to +1. If there is a strong negative linear relationship between the variables, the correlation r is close to -1. When two variables have weak or no association, the correlation r is close to 0.

a positive linear relationship : r > 0

a negative linear relationship : r < 0

The absolute value of $r$ tells the strength of a relationship and the sign indicate the direction.

Aspect Simple Linear Regression Correlation

Key Output Coefficients $\beta_0$ (intercept) and $\beta_1$ (slope) for prediction. Correlation coefficient $r$ (often Pearson’s $r$), ranging from $-1$ to $+1$.

Role in Machine Learning A supervised learning algorithm used for predictive modeling. A statistical tool for feature selection and exploratory data analysis.

3.2 Performance measure for a predictive model¶

Regression:¶

  1. Mean Absolute Error (MAE): The average of the absolute differences between predicted and actual values.
$$MAE=\frac{1}{n}\sum_{i=1}^{n} |y_i-\hat{y}_i|$$
  1. Mean Squared Error (MSE): The average of the squared differences between predicted and actual values.
$$MSE=\frac{1}{n}\sum_{i=1}^{n}(y_i-\hat{y}_i)^2$$
  1. Root Mean Squared Error (RMSE): The square root of the MSE. It's in the same units as the response and provides a more interpretable measure of error.
$$RMSE = \sqrt {MSE} = \sqrt { \frac{1}{n}\sum_{i=1}^{n}(y_i-\hat{y}_i)^2 } $$
  1. R-squared : A measure of how well the model explains the variance in the response. It ranges from 0 to 1, with a higher value indicating a better fit.

R-squared = $r^2$
where $r$ is the correlation between the predicted values and the true observed values.

It's beneficial to compute both the Mean Square Error and the R-squared, as each metric provides distinct insights. MSE reveals the average deviation between the predicted values generated by the model and the actual values. Conversely, $R^2$ informs us about the effectiveness of the predictor variables in accounting for the variability in the response variable.

For example: if R-squared is 0.83, about $83\%$ of the information (varition) in the response can be predicted (interpreted) by the model.

  1. Median Absolute deviation (MAD): The median of the absolute differences between predicted and actual values.

    The median of all $ |y_i-\hat{y}_i | $, $i = 1, 2, \cdots, n$.

Classification:¶

The primary objective is to predict a target variable that takes on discrete values (classes or categories). To assess the performance, these metrics include:

  1. Classification Accuracy:
    This is the proportion of correctly predicted instances out of the total instances. It is the simplest and most intuitive metric, but it can be misleading, especially when dealing with imbalanced datasets.

$$ \text{Accuracy} = \frac{\text{Number of Correct Predictions}}{\text{Total Number of Predictions}} $$

  1. Logarithmic Loss (Log Loss):
    Log loss measures the uncertainty of predictions by penalizing confident and incorrect classifications more heavily. It provides a measure of how close the predicted probabilities are to the actual class labels, where lower values indicate better performance.

    $$ \text{Log Loss} = - \frac{1}{N} \sum_{i=1}^{N} \left[ y_i \log(p_i) + (1 - y_i) \log(1 - p_i) \right]

$$ where (y_i) is the actual class label and (p_i) is the predicted probability for that class.

  1. Area Under the Curve (AUC):
    The Area Under the Receiver Operating Characteristic (ROC) Curve (AUC-ROC) measures the model’s ability to distinguish between the classes. A higher AUC indicates a better performance, with 1 being perfect classification and 0.5 indicating random guessing.

  2. F1 Score:
    The F1 score is the harmonic mean of precision and recall. It is useful when you need to balance the trade-off between precision (positive predictive value) and recall (sensitivity), especially in cases of class imbalance.

$$ F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} $$

  1. Precision:
    Precision measures how many of the predicted positive instances are actually positive. It is a useful metric when the cost of false positives is high.

$$ \text{Precision} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Positives}} $$

  1. Recall (Sensitivity or True Positive Rate):
    Recall measures how many actual positive instances the model is able to identify. It is important when the cost of false negatives is high.

$$ \text{Recall} = \frac{\text{True Positives}}{\text{True Positives} + \text{False Negatives}} $$

  1. Confusion Matrix:
    A confusion matrix provides a detailed breakdown of the model’s predictions by showing the counts of true positive (TP), true negative (TN), false positive (FP), and false negative (FN) predictions. It serves as the foundation for other metrics and provides a comprehensive view of the classification performance.

3.3 Variants of a simple linear regression¶

Polynomial regression¶

Polynomial regression is a type of regression in machine learning for modeling the relationship between a the response and one or more predictors by fitting a polynomial equation to the data. Unlike simple linear regression, polynomial regression can capture more complex and non-linear relationships between variables.

$$Y_i = \beta_0 + \beta_1 X_i + \beta_2 X_i^2 + \ldots + \beta_k X_i^k + \varepsilon_i$$

R codes: fit = lm(y ~ poly(x , k , raw=T), data= …)

Median regression¶

  • Mean Absolute Deviation (error) (minimizing $MAD=\sum_{i=1}^{n}|y_i-\hat{y}_i|$ )

In median regression, the criterion is to minimize the sum of the absolute residuals. This means the model finds the line that best fits the data by reducing the total absolute differences between predicted and actual values, estimating the conditional median of the response based on the predictor variable. Unlike least squares regression, which minimizes squared residuals, median regression focuses on absolute differences.

Median regression ( 0.5 quantile regression ) is a regression method to estimate the relationship between variables in a dataset. It focuses on the median (50th percentile) of the response. It is an alternative to ordinary least squares (OLS) regression to estimate the conditional mean of the dependent variable. It is robust to outliers and provides another measure of central tendency not influenced by outliers.

Median regression estimate the model parameters via optimization techniques minimizing the sum of absolute deviations (also known as $L_1$ loss) between the observed and predicted values. The estimated coefficients in median regression represent changes in the median of the response for a one-unit change in the predictor.

Overfitting¶

Overfitting is one issue in predective modeling. It occurs when a model learns the data too much and captures pure random fluctuations rather than just the underlying patterns or relationships. An overfit model performs very well on the training data but fails to generalize effectively to new data.

Key characteristics of overfitting in machine learning include: high training accuracy, poor generalization, model complexity, high variance, fitting irrelevant patterns, ....

The simplest way to prevent overfitting is to start with a small model. To overcome overfitting, it's important to use techniques such as cross-validation, regularization, model selection, ...

Data spliting: Training and test datsets¶

One major challenge in predictive modeling is to make it work accurately on new data. Training and test datsets are essential concepts in preditive modeling and machine learning, which are crucial for developing and evaluating predictive models. To check the model's performance, it's crucial to test it against data points that were not part of its training set. These unseen data points essentially serve as a test for the model's accuracy. Among the most reliable methods for assessing a machine learning model's effectiveness is the Cross-validation technique. Cross-validation involves setting aside a portion of the dataset for testing, separate from the training data. After building the model, this reserved data is employed to assess its performance as the testing data. During testing, the model predicts values for the dependent variable, and accuracy is measured by comparing these predictions to the actual values.

Training data: The training data is a subset of your dataset that is used to train a model.

Test data: The test data is a distinct portion of your data that is not used during the training phase. It serves as an independent dataset that the model has never seen before. The main purpose of the test set is to assess how well the model generalizes to new, unseen data. After the model has been trained on the training set, you use the test set to evaluate its performance.

The size of the training and test data sets can different depending on your dataset/problem, but a common split is 70-80% for training and 20-30% for test data. You might train and evaluate multiple models using the same test set to choose the one with the best performance. Proper separation of data into training and test data sets helps ensure that your model's performance estimates are realistic and unbiased, reflecting how it will perform on new, unseen data in real-world scenarios.

Cross-validation¶

Cross-validation is a fundamental technique to guarantee that the predictive models are robust and capable of generalizing to new unseen data. It involves resampling the dataset multiple times, allowing the model to be fitted on different subsets. This process helps assess how well the model is likely to perform in practice. Additionally, cross-validation can be an effective method for detecting overfitting. If a model performs significantly better on the training set than on the test (validation) set, it may indicate overfitting.

Commonly used cross-validation methods include:

1. Leave-One-Out Cross-Validation (LOOCV):¶

This method tests the model by leaving out one data point at a time and using the rest for training. It's great for small datasets or when accuracy matters most, but it can be slow for big datasets.

If there are $n$ data points in the dataset, LOOCV needs to fit the model $n$ times.

2. k-Fold Cross-Validation¶

This is a common approach where you split your data into "k" equal parts or "folds." You train and test the model "k" times, with each fold taking turns as the test set and the others as the training set. It gives you a more reliable measure of how well your model works. Leave-One-Out is a special case of this when "k" equals the number of data points.

See plot at: https://scikit-learn.org/stable/modules/cross_validation.html

3. Repeated K-Fold Cross-Validation¶

Repeated K-Fold extends K-Fold by repeating the cross-validation process multiple times ($p$ times) with different splits in each iteration. This is useful when we want to add more robustness to the evaluation by testing on multiple random data splits.

  • The dataset is split into k folds in each repetition.
  • The process is repeated p times, producing p different k-fold splits.
  • The final performance is averaged across all p repetitions.

This method helps in reducing the variance of the evaluation by testing the model on different data splits.

4. Leave-P-Out Cross-Validation (LPO)¶

Leave-P-Out Cross-Validation is a generalization of Leave-One-Out where instead of leaving out one sample, p samples are left out in each iteration.
