Chapter 5.4 Support vector machine (SVM)¶

A Support Vector Machine (SVM) is a supervised machine learning algorithm used for both classification and regression tasks. The core idea of an SVM is to find an optimal hyperplane that best separates data points belonging to different classes in the feature space. In the context of classification, SVMs are particularly effective for problems where data points are not linearly separable, that is, where no single straight line or plane (in higher dimensions) can cleanly divide the classes.

The support vectors in an SVM are the data points that lie closest to the decision boundary (the hyperplane) separating the classes. These points are critical because they define and determine the optimal hyperplanedirectly influencing its position and orientation. SVMs are capable of handling both linear and nonlinear classification tasks. For nonlinear problems, SVMs employ a technique known as the kernel trick. A kernel function computes the inner product of two data points in a higher-dimensional feature space without explicitly transforming the data into that space. This elegant approach allows SVMs to model complex, nonlinear decision boundaries efficiently.

(source: https://xavierbourretsicotte.github.io/Kernel_feature_map.html)

For example, a simple application of the kernel trick involves transforming the original variables, x and y, into a new feature space by introducing an additional dimension, z. This transformation allows the data to be visualized in three dimensions, where the two classes that were previously overlapping in the 2D space become more easily separable. In this 3D space, a clear margin between the two classes emerges, making it possible to define an optimal hyperplane that effectively separates them.

The primary objective of an SVM is to identify this hyperplane that maximizes the margin: the distance between the hyperplane and the nearest data points from each class (the support vectors). A larger margin generally leads to better generalization, enabling the model to classify new, unseen data more accurately and robustly.

SVM regression¶

Support Vector Regression (SVR), also known as Linear SVM Regression, is a machine learning technique derived from the Support Vector Machine (SVM) framework. While traditional SVMs are primarily used for classification, SVR extends the same principles to regression problems to predict continuous numerical values rather than categorical labels. The goal of SVR is to find a linear function that best approximates the relationship between input features and the target variable while maintaining a balance between model simplicity and prediction accuracy. In other words, SVR seeks a function that deviates from the observed targets by no more than a specified margin, while remaining as flat as possible.

The optimization problem for Linear Support Vector Regression (SVR) can be written as:

$$ \text{Minimize: } J(w, b) = \frac{1}{2} \|w\|^2 + C \sum_{i=1}^n (\xi_i + \xi_i^*) $$

subject to:

$$ y_i - (w^\top x_i + b) \leq \varepsilon + \xi_i, $$$$ (w^\top x_i + b) - y_i \leq \varepsilon + \xi_i^*, $$$$ \xi_i, \, \xi_i^* \geq 0. $$

Here:

  • $w$ is the weight vector defining the slope of the regression function.
  • $b$ is the bias term (intercept).
  • $\|w\|^2 / 2$ is the regularization term that keeps the model flat.
  • $\xi_i$ and $\xi_i^*$ are slack variables that measure deviations larger than $\varepsilon$.
  • $C$ is a regularization parameter controlling the trade-off between model complexity and tolerance for errors.
  • $\varepsilon$ defines the insensitive margin within which prediction errors are ignored.

Tuning Parameter in SVM¶

In Support Vector Machines (SVMs), the cost parameter (commonly denoted as C) is a key hyperparameter that controls the trade-off between achieving a low training error and maintaining good generalization to unseen data.
It essentially determines how strictly the SVM penalizes misclassified (or, in regression, mispredicted) data points during training.

  • When C is small:
    The model allows for a wider margin and tolerates more misclassifications or deviations from the margin.
    This encourages a simpler, smoother decision boundary and helps prevent overfitting though it may lead to underfitting if the margin becomes too loose.

  • When C is large:
    The model places greater emphasis on correctly classifying all training samples, even at the cost of a smaller margin.
    This often yields a more complex model that fits the training data closely, but it may overfit and perform poorly on unseen data.

The parameter C governs the biasvariance trade-off in SVMs:

  • A small C higher bias, lower variance (simpler model).
  • A large C lower bias, higher variance (complex model).

Thus, tuning C properly is essential to balance model flexibility and generalization performance.

In practice, the optimal value of C is chosen through model validation techniques such as:

  • Cross-validation: Evaluate model performance for several C values and select the one minimizing validation error.
  • Grid search: Systematically explore combinations of hyperparameters (e.g., C and kernel parameters).
  • Automated optimization tools: Functions like tune() in R or GridSearchCV() in Python streamline this process.

A well-chosen C ensures that the SVM model generalizes well and performs reliably on unseen data.

Hyperparameter Tuning in SVM with RBF Kernel, using train function¶

When training a Support Vector Machine (SVM) with a Radial Basis Function (RBF) kernel, two key hyperparameters must be tuned to achieve the best model performance:

  • C (Cost parameter)
  • (Sigma, or gamma-related parameter)

These hyperparameters control the models flexibility, smoothness of the decision boundary, and tolerance for misclassification or error.

Scaling (also called standardization or normalization) is strongly recommended when using Support Vector Machines (SVMs), especially with RBF, polynomial, or sigmoid kernels. Because SVMs are distance-based algorithms, features with larger numerical ranges can dominate the distance metric, leading to biased results.
Moreover, kernel functions are highly sensitive to the magnitude of feature values. Without scaling, the kernel may produce distorted similarity measures, which can degrade model performance. The optimization problem solved by SVM (particularly when using a large cost parameter ( C ) or a small kernel width ( \sigma )) can also become ill-conditioned when features vary widely in scale. This can result in slower convergence or suboptimal solutions during training.

In practice, features should typically be:

  • Standardized (mean = 0, standard deviation = 1), or
  • Normalized (rescaled to [0, 1]).
